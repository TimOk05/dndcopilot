# 📋 ИНСТРУКЦИИ ДЛЯ ПРОЕКТА D&D COPILOT

## 🎯 ОСНОВНЫЕ ПРИНЦИПЫ

### 1. **AI ВСЕГДА ВКЛЮЧЕН ПО УМОЛЧАНИЮ**
- ✅ **AI используется по умолчанию** во всех генераторах
- ✅ **При недоступности AI** показывается ошибка (НЕ fallback)
- ✅ **Никаких переключателей AI** в интерфейсе
- ✅ **AI чат работает только с AI** - показывает ошибку без AI

### 2. **ПОЛИТИКА NO_FALLBACK**
- 🚫 **ЗАПРЕЩЕНО создавать fallback системы**
- 🚫 **ЗАПРЕЩЕНО использовать резервные данные**
- ✅ **Показывать ошибки** при недоступности API
- ✅ **Четкие сообщения** с инструкциями по исправлению

### 3. **ТЕСТИРОВАНИЕ**
- 🧪 **Для локального тестирования** создавайте тестовые файлы БЕЗ AI
- 🧪 **Используйте `use_ai=off`** в тестовых запросах
- 🧪 **Проверяйте ошибки** вместо fallback данных

## 📝 ПРИМЕРЫ КОДА

### ✅ **Правильно - AI всегда включен:**
```php
$use_ai = true; // AI всегда включен
$params = [
    'race' => $race,
    'class' => $class,
    'use_ai' => 'on'
];
```

### ✅ **Правильно - показываем ошибку:**
```php
if (!$result['success']) {
    return [
        'success' => false,
        'error' => 'AI API недоступен',
        'message' => 'Проверьте подключение к интернету'
    ];
}
```

### ❌ **Неправильно - fallback:**
```php
// НЕ ДЕЛАЙТЕ ЭТОГО!
if (!$result['success']) {
    return $this->generateFallbackData();
}
```

### ❌ **Неправильно - переключатель AI:**
```html
<!-- НЕ ДОБАВЛЯЙТЕ ЭТОГО! -->
<select name="use_ai">
    <option value="on">Включено</option>
    <option value="off">Отключено</option>
</select>
```

## 🧪 ТЕСТИРОВАНИЕ

### **Для локального тестирования:**
```php
// Используйте use_ai=off для тестов
$post_data = "action=generate_character&race=human&class=fighter&use_ai=off";
```

### **Проверка ошибок:**
```php
if ($data && $data['success']) {
    echo "✅ УСПЕШНО\n";
} else {
    echo "❌ ОШИБКА: " . ($data['message'] ?? 'Неизвестная ошибка') . "\n";
}
```
